<details>
<summary><b>🏥 Medical Care — AI-Powered Doctor Appointment Platform</b></summary>

<br/>

A production-grade healthcare booking platform with three distinct panels — **User, Doctor, and Admin** — built on the MERN stack with integrated payments, cloud media management, and an AI diagnostic assistant.

| Attribute | Details |
|:---|:---|
| **Stack** | React · Node.js · Express · MongoDB · Razorpay · Cloudinary · Groq AI |
| **Scale** | 3-panel architecture (User / Doctor / Admin) with role-based access control |
| **Performance** | Optimised query pipelines · Lazy loading · Cloudinary CDN for media |
| **Security** | JWT authentication · Bcrypt hashing · Role-guard middleware |
| **AI Feature** | Groq-powered LLaMA 3 chatbot for symptom triage and medical Q&A |
| **Impact** | Live production deployment · Full appointment lifecycle management |
| **Repository** | [medical-care-frontend](https://github.com/rimsha249/MEDICAL-CARE) |

Admin panel provides complete platform oversight including doctor verification and analytics.

🔗 **Live:** [medical-care-frontend-two.vercel.app](https://medical-care-frontend-two.vercel.app) · [medical-care-admin.vercel.app](https://medical-care-admin.vercel.app)

<details>
<summary><b>📊 Store Intelligence System — Real-Time CCTV Analytics Dashboard</b></summary>

<br/>

A computer vision pipeline built for the **Purplle Tech Challenge 2026 Round 2** that processes live CCTV footage to extract retail intelligence — customer footfall, zone heatmaps, and dwell-time metrics — served through a live FastAPI dashboard.

| Attribute | Details |
|:---|:---|
| **Stack** | FastAPI · YOLOv8 · Docker · SQLite · Python · Render |
| **Scale** | Real CCTV footage processing · Multi-zone tracking |
| **Performance** | YOLOv8 inference pipeline · Async FastAPI endpoints · SQLite for metric storage |
| **Security** | Containerised deployment · Environment-isolated config |
| **AI Feature** | YOLOv8 object detection for person tracking and zone classification |
| **Impact** | Deployed to Render · Built under hackathon conditions · Production-ready |
| **Repository** | [store-intelligence-system](https://github.com/rimsha249) |

**Architecture highlights:** YOLOv8 runs frame-level inference on CCTV video streams; detected bounding boxes are mapped to predefined store zones; aggregated metrics (footfall count, average dwell time, peak zone activity) are persisted in SQLite and exposed via FastAPI REST endpoints consumed by a live analytics dashboard.

</details>

<details>
<summary><b>🚦 Smart Traffic Management System — Award-Winning Hardware + AI</b></summary>

<br/>

An AI-driven adaptive traffic signal control system combining Arduino microcontrollers with YOLOv8 computer vision for real-time vehicle detection and dynamic signal timing. Awarded **1st Prize** at the institutional level.

| Attribute | Details |
|:---|:---|
| **Stack** | Arduino · YOLOv8 · Python · OpenCV · Serial Communication |
| **Scale** | Multi-intersection control with per-lane vehicle counting |
| **Performance** | Real-time YOLOv8 inference feeding signal timing decisions under 200ms |
| **Security** | Hardware-isolated signal control · Fail-safe default states |
| **Impact** | 🥇 1st Prize · Demonstrates hardware-software-AI integration |
| **Repository** | [GitHub](https://github.com/rimsha249) |

**Architecture highlights:** YOLOv8 model detects and counts vehicles per lane from a camera feed; counts are serialised and sent to Arduino via USB serial; Arduino logic dynamically adjusts green-signal duration proportional to lane density, reducing average wait time versus fixed-cycle systems.

</details>
